# 43B3TZ Sports Intelligence platform: AI & ML Architecture Guide

This document details the high-assurance engineering, mathematical, and architectural decisions behind the **Enterprise AI Intelligence Platform** designed and implemented for the 43B3TZ system.

---

## 1. AI Provider Platform (`/server/ai/`)

The core AI infrastructure centers on an exchangeable, multi-provider architectural model. 

### Core Components
*   **`ProviderManager` (`server/ai/provider.ts`)**: Orchestrates unified requests to Google Gemini, OpenAI, Anthropic, NVIDIA, Ollama, and OpenRouter. It supports transparent in-memory response caching, automated API cost tracking, and automatic provider-fallback.
*   **`IAIProvider` (`server/ai/types.ts`)**: Establishes a standard contract for token streaming, content generation, and structured outputs.

### Provider Matrix and Configuration
*   **Google Gemini**: Primary provider using the modern `@google/genai` SDK and the target `gemini-3.5-flash` model (using the platform-mandated `'aistudio-build'` User-Agent).
*   **Ollama**: Local container-authoritative execution (utilizing zero-cost routing, falling back automatically if cloud API rate limits are reached).
*   **NVIDIA & OpenRouter**: Failover configurations requiring standard metadata headers (`HTTP-Referer` and `X-Title`).

## 2. Prompt Management & Versioning

Prompt design in production Sports AI must avoid prompt drift. 

### Immutable Versioning
The `PromptRegistry` (`server/ai/registry.ts`) manages prompts as immutable code assets:
*   Prompts are registered with unique `id` and `version` fields.
*   Attempts to register duplicate versions throw a strict configuration error.

### A/B Testing Routing
```typescript
const prompt = promptRegistry.getABPrompt("explainable_prediction", userId);
```
Determines prompt selection dynamically by hashing the unique `userId` to route 90% of requests to the stable control prompt (v1.0.0) and 10% to experimental challenger templates (v1.0.1) for live quality evaluation.

## 3. The 13-Stage Prediction Pipeline (`/server/predictions/pipeline.ts`)

Predictions do not execute in a black-box. The pipeline enforces a strictly structured, performance-monitored 13-stage progression:

```
[1. Data Collection] ➔ [2. Validation] ➔ [3. Cleaning] ➔ [4. Feature Engineering] 
  ➔ [5. Feature Selection] ➔ [6. Model Selection] ➔ [7. Inference] 
  ➔ [8. Confidence Calibration] ➔ [9. Risk Analysis] ➔ [10. Kelly Optimization] 
  ➔ [11. Recommendation] ➔ [12. Post-Match Evaluation]* ➔ [13. Continuous Learning]*
```
*\*Stages 12 and 13 run asynchronously as soon as a fixture result is ingested by the ETL platform.*

## 4. Feature Engineering (`/server/predictions/features/`)

Features snapshot tactical, mental, physical, and market factors:
*   **Elo Differentials**: Mathematical team strength differentials.
*   **Form Momentum**: Weighted moving-average over the past 5 matches.
*   **xG Differentials**: True expected goals creation vs concession indices.
*   **Market movement (Bookmaker Drift)**: High-frequency odds speed indicators capturing insider and smart money activity.

### Feature Selection
Features are filtered dynamically: only features with an `importanceScore >= 0.40` are fed into active downstream inference matrices.

---

## 5. Model Registry & Deployment Roles (`/server/predictions/registry/`)

Models reside under a state-machine registry containing strict production roles:
*   `champion`: Primary model routing live production traffic.
*   `challenger`: Active online benchmark comparing accuracy metrics.
*   `shadow`: Receives dual-write predictions silently in the background for validation checks.
*   `fallback`: Static baseline logistic model ensuring graceful degradation.
*   `canary` & `experimental`: Locked behind specific test cohorts.

---

## 6. Confidence Calibration (`/server/predictions/calibration/`)

Raw logits generated by tree-based models (such as LightGBM or XGBoost) are not true probability scores. The platform applies **Platt Scaling**:

$$P(\text{Outcome} | x) = \frac{1}{1 + \exp(A \cdot f(x) + B)}$$

This maps uncalibrated logits into exact probability curves. Accuracy and reliability scores are calculated to track the quality of the calibration continuously.

---

## 7. Model Evaluation Suite (`/server/predictions/evaluation/`)

Evaluations are computed over predictions and betting records:
*   **Log Loss**: Penalizes overconfident wrong predictions.
*   **Brier Score**:
    $$\text{BS} = \frac{1}{N} \sum_{i=1}^N \sum_{j=1}^C (p_{ij} - y_{ij})^2$$
*   **Expected Calibration Error (ECE)**: Groups predictions into 10 confidence bins and computes the weighted average of confidence-accuracy differentials.
*   **Betting Return (ROI & Yield)**: Actual net-profit divided by total staked bankroll.
*   **Closing Line Value (CLV)**: Measures beat-rate against bookmaker closing lines:
    $$\text{CLV} = \frac{\text{Opening Odds}}{\text{Closing Odds}} - 1$$

## 8. Historical Backtesting Platform (`/server/predictions/backtesting/`)

The `BacktestingPlatform` (`server/predictions/backtesting/backtestPlatform.ts`) simulates historical seasons across:
*   Premier League
*   La Liga
*   Serie A
*   Bundesliga
*   Champions League
*   World Cup

It replays chronological matchups, feeds telemetry into the prediction pipeline, executes a fractional Kelly betting strategy, and generates full bankroll timelines tracking **Maximum Drawdown (MDD)** and **Yield**.

## 9. Research Lab (`/server/predictions/research/`)

The ML Research Lab (`server/predictions/research/researchLab.ts`) logs every pipeline experiment:
*   Tracks dataset splits, active feature indices, model hyper-parameters, and LLM prompts.
*   Supports one-click **Model Promotion** (re-assigning champion roles).
*   Enforces automated **Rollback Capabilities** to baseline fallbacks if champion calibration error exceeds critical bounds.

---

## 10. Explainable AI (`/server/predictions/explainability/`)

The `XAIEngine` (`server/predictions/explainability/xaiEngine.ts`) translates complex features into human-readable narratives:
*   Analyzes ELO, form, and xG feature values to output logical **Confidence Drivers**.
*   Highlights high-friction **Risk Factors** (e.g. weather conditions, injuries, or market volatility).
*   Uses Google Gemini's `gemini-3.5-flash` to write personalized tactical reports.

---

## 11. Continuous Learning (`/server/predictions/learning/`)

The continuous learning feedback loop operates after every completed match:
1.  Ingests actual results and closing bookmaker odds.
2.  Computes instantaneous Log Loss and Brier Score.
3.  Performs **Online Calibration Refinement** (tuning Platt scaling intercept offsets).
4.  Updates model rankings; if a challenger outperforms the champion over a moving average threshold, it triggers automated promotion.
5.  Updates prompt quality evaluation scores based on explainability accuracy.
